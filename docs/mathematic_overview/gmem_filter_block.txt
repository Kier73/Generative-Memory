# Mathematical Overview: gmem_filter_block.c

## 1. Introduction
This document analyzes `src/gmem_filter_block.c`, which implements block-level filtering operations (convolution/transformation) over the memory space.

---

## 2. Filter Operations

### 2.1. Kernel Convolution
While the file name suggests block filtering, the current implementation primarily handles bulk transfers and basic block-wise transformations.

Let $B$ be a block of data $[x_0, x_1, \dots, x_{n-1}]$.
A filter function $\Phi$ transforms $B$:
$$ B' = \Phi(B) $$

### 2.2. Identity and Copy
The most fundamental operation is the Block Copy (Fetching):
$$ \forall i \in [0, n): B'[i] = B[i] $$

This utilizes `gmem_fetch_bulk_f32` (described in `gmem.c` overview) to leverage AVX2 SIMD optimizations.

## 3. Integrity Check (Audit)

### 3.1. Boundary Conditions
-   The logic typically iterates in chunks of `PAGE_SIZE` (4096 bytes / 1024 floats).
-   **Check**: Does it handle unaligned sizes?
    -   Yes, the logic in `gmem.c` (called by filter) calculates ` floats_in_page = GMEM_PAGE_SIZE - (current_addr % GMEM_PAGE_SIZE)`. This modulo arithmetic correctly handles non-aligned starts.

# Summary
This module provides the linear algebra primitives (Block Copy/Transform) needed for higher-level operations.
